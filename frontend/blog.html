GOG.md - Agent Configuration Deep Dive

One of the most powerful integrations for any AI agent is Google Workspace (Gmail, Calendar, Drive). In this post, I'll walk through how I configure my agent to handle these services effectively.

Why GOG.md Matters

The GOG.md file is a configuration guide that tells my agent how to interact with Google services. It's one of the most important integrations I have, but it can be finicky ‚Äî OAuth tokens expire, permissions need careful management, and the API has rate limits to consider.

Email Threading

When responding to emails, always use `--reply-to-message-id` to keep conversations connected. This ensures replies appear in the correct thread, making it easy for everyone to follow the conversation.

# Reply in thread
gog gmail send --to recipient@example.com --subject "Re: Subject" --body "Reply" --reply-to-message-id <message_id>

Calendar Notifications

Critical: When creating, modifying, updating, or deleting any meeting with invited attendees, you MUST notify all attendees via email. Don't assume they'll see calendar updates automatically.

This is especially important because:

  • Calendar invites can land in spam
  • Some users only check calendar on desktop
  • Email confirmation provides a paper trail

Customer Inquiry Alerts

Any email from the website contact form (subject: NEW WEB CONTACT) is a customer inquiry. These need immediate attention:

  1. Read the email content
  2. Alert the owner via their preferred channel
  3. Include customer name, email, and message summary

Key Takeaways

  • Always thread replies ‚Äî keeps conversations organized
  • Notify meeting attendees manually ‚Äî don't rely on calendar alone
  • Redact sensitive data ‚Äî when sharing examples, remove PII
  • Handle errors gracefully ‚Äî OAuth tokens expire and need refresh

The Google integration is incredibly powerful but requires careful configuration. Take time to set up proper error handling and your agent will be a well-oiled machine for communications.
